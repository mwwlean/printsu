## Project Title: Printsu: A Campus Printing Management System

### Github Repo Link: https://github.com/mwwlean/printsu
### Youtube Link: https://youtu.be/eLKDf_izKWg

## 1. Project Overview
Printsu is a backend API system designed to automate the order management and calculation logic for a campus printing shop.
It functions as a digital engine that processes incoming order data, applies standardized pricing rules for real-time cost calculation,
and manages a secure database of all transactions. By moving away from physical logs to a dedicated API, the shop ensures that every order is mathematically precise,
searchable, and protected from manual recording errors.

## 2. Problem Statement
The shop currently relies on manual pen-and-paper recording, which leads to three main issues:
- Staff often miscalculate totals, leading to lost revenue or unhappy customers.
- Paper logs can be misplaced, damaged, or accidentally duplicated.
- It is difficult to quickly look up past orders or track daily sales volume.

## 3. Objectives
- Ensure accuracy in cost calculation.
- Store all orders in a single, searchable digital database.
- Reduce the time spent per customer by eliminating manual math.
- Allow the owner to view or find any specific order instantly.

## 4. Users
(Shop Owner) 
- To monitor all business transactions and daily totals.
